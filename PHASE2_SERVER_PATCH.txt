WHAT THIS DOES
--------------
Right now `resolveSystemPrompt()` looks for a key like `fr:politics`, never
finds one, and silently falls back to `fr:default`. That is why the topic
parameter currently does nothing.

This patch adds the five new French topic keys. The moment you deploy it,
the five new pages stop using the generic prompt and start using a prompt
written for their subject — with NO change to any HTML. The slugs already
match.

The `fr:default` prompt stays exactly as-is and remains the fallback.

HOW TO APPLY
------------
In the SYSTEM_PROMPTS object, paste the five entries below immediately
AFTER the existing `'fr:default'` entry and BEFORE `'en:default'`.
Nothing else in the file changes.

WHEN YOU MIRROR TO ENGLISH
--------------------------
The slugs are language-neutral and already shared. To light up the English
pages later, add the same five keys with the `en:` prefix:

    'en:culture', 'en:food-travel', 'en:environment',
    'en:art-philosophy', 'en:history'

and for the bilingual course:

    'bilingual:culture', 'bilingual:food-travel', 'bilingual:environment',
    'bilingual:art-philosophy', 'bilingual:history'

No HTML changes needed on that side either — same pattern.

VERIFY AFTER DEPLOY
-------------------
Open frenchc1.kingcretot.com/histoire.html, start the conversation, and ask
the tutor a question that a generic prompt would answer blandly (e.g.
"Pourquoi la Révolution divise-t-elle encore les Français ?"). A tuned prompt
will engage with the competing interpretations. That's the tell.
